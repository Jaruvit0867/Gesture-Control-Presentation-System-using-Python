3. **กราฟเปรียบเทียบ** ความแม่นยำแต่ละฟังก์ชัน

4. **อภิปรายผล** สาเหตุของ false detection

#### ข้อจำกัดของระบบ (System Limitations)

1. **ด้านแสง**: ต้องการแสงสว่างอย่างน้อย 150 lux เพื่อความแม่นยำสูงสุด
2. **ระยะห่าง**: ทำงานได้ดีที่ 50-80 ซม. ประสิทธิภาพลดลงเมื่อเกิน 100 ซม.
3. **ท่าทาง 2 นิ้ว (Drag)**: มีความสับสนบ่อยเมื่อนิ้วชี้และนิ้วกลางอยู่ใกล้กันเกินไป
4. **การตรวจจับนิ้วโป้ง**: ต้องหันมือให้ถูกต้อง (ฝ่ามือหันเข้าหากล้อง)
5. **Multi-person detection**: ระบบอาจสับสนเมื่อมีมือหลายคนในเฟรม

## 4. 📐 รายละเอียดเชิงเทคนิค (Algorithm Details)

### ❌ ปัญหา
- ไม่ระบุ threshold และ logic การตัดสินใจ
- ภาพ architectural diagram ขาดรายละเอียด

### ✅ สิ่งที่ต้องเพิ่ม

1. **Gesture Classification Logic** (จากโค้ดจริง):

```
การตรวจจับนิ้ว:
- นิ้วโป้ง: เปรียบเทียบแกน X (tip vs. pip) แยกมือซ้าย/ขวา
- นิ้วอื่นๆ: เปรียบเทียบแกน Y (tip < pip = ยก)

Threshold ในระบบ:
- min_detection_confidence: 0.7
- min_tracking_confidence: 0.6
- swipe_dist_threshold: 0.15 (normalized)
- swipe_timeout: 0.5 วินาที
- swipe_cooldown: 0.6 วินาที

Mouse Smoothing:
- smooth_factor: 5.0 (ค่าเริ่มต้น)
- Adaptive range: 2.0 - 10.0 (ปรับตาม FPS)
```

2. **Data Flow Diagram ควรแสดง**:

```
Camera Frame → BGR to RGB → MediaPipe Hands → 
Hand Landmarks (21 points) → Finger Counting → 
Gesture State Machine → Mouse/Keyboard Action
```
